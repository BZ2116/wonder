# v0.1.0 Windows Release Design

## Goal

Ship Wonder v0.1.0 as a Windows-first Tauri desktop release that can be downloaded from GitHub Releases, installed through an NSIS setup executable, and reused as a simple release process for future personal use and portfolio demonstration.

## Scope

This release focuses on the basic packaging and version-management flow. It does not include in-app update checks, updater signatures, auto-update manifests, or code-signing certificates. Those can be added after v0.1.0 is downloadable and the repository presentation is stable.

## Repository And Versioning

The target GitHub repository is `https://github.com/BZ2116/wonder.git`.

The release version is `0.1.0`, represented consistently in:

- `package.json`
- `package-lock.json`
- `src-tauri/Cargo.toml`
- `src-tauri/tauri.conf.json`

The Git tag for the release is `v0.1.0`. The GitHub Release title should be `Wonder v0.1.0`.

## Packaging

Use the official Tauri bundler with the Windows NSIS target. The release artifact should be the Tauri-generated setup executable from `src-tauri/target/release/bundle/nsis/`.

The existing hand-written `installer.nsi` is not part of the primary v0.1.0 release flow. It can remain in the repository for now, but release documentation should direct maintainers to `npm run tauri:build`.

## Documentation

Add release documentation that explains:

- Required local tools: Node.js, Rust, and the Tauri Windows build prerequisites.
- Verification commands before release.
- How to build the Windows installer.
- Where to find the generated setup executable.
- How to create and push `v0.1.0`.
- How to publish the GitHub Release manually.
- That in-app updates are intentionally deferred.

The README should expose the download and build path clearly enough for a recruiter or future self to understand how the app is packaged.

## Validation

Before calling v0.1.0 ready, run:

- `npm run test`
- `npm run build`
- `npm run tauri:build`

If the full Tauri build is blocked by local machine prerequisites, record the exact blocker and still leave the release instructions correct.
