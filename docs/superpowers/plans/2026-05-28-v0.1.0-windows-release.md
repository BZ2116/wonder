# v0.1.0 Windows Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prepare Wonder v0.1.0 for a Windows NSIS installer release on GitHub.

**Architecture:** Keep the release pipeline simple and local-first: align app versions, configure Tauri to produce only the Windows NSIS installer, document the manual GitHub Release flow, and verify with the existing test/build commands. In-app updates are documented as deferred work rather than implemented in this release.

**Tech Stack:** Tauri 2, Vue 3, TypeScript, Vite, Rust, NSIS bundling, GitHub Releases.

---

## File Structure

- `package.json`: npm package metadata and scripts; version must be `0.1.0`.
- `package-lock.json`: lockfile metadata; root package version must match `0.1.0`.
- `src-tauri/Cargo.toml`: Rust package metadata; version must match `0.1.0`.
- `src-tauri/tauri.conf.json`: Tauri product metadata and bundler target; version must match `0.1.0` and bundle target should be `nsis`.
- `.gitignore`: prevents local runtime data and workspace helper directories from being committed to the release repository.
- `README.md`: public project entry point; add download/build/release notes for portfolio readers.
- `docs/release.md`: maintainer release checklist for v0.1.0 and future manual releases.
